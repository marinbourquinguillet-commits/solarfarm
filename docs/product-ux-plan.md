# Solar Farm Operations Hub – Product + UX Plan (MVP)

## 1) Information architecture
- Login
- Home Dashboard (role-based big buttons)
- Timesheets
- Vehicle Pre-starts
- Sites & Maps
- Documents
- Reports (Incidents/Issues)
- Messaging & Groups
- Daily Team Board
- Team Daily Production Reports
- Supervisor tools

## 2) Navigation (mobile-first)
- Top area: user + role + site quick selector.
- Main area: large action buttons.
- Context panel/form below button tap.
- Supervisor sees extra buttons.

## 3) Key screens
1. Home / Dashboard
2. Timesheet start/end form
3. Timesheet history
4. Timesheet approval queue
5. Vehicle pre-start form
6. Vehicles needing repair
7. Site list + site detail
8. Document library with search/filter
9. Issue report create form
10. Incident list and status update
11. Messaging channels + thread
12. Daily board view
13. Team production report form + totals list

## 4) Core daily user flows
### Worker
1. Open app from phone (or scan QR).
2. Start shift.
3. Complete vehicle pre-start if assigned.
4. View today’s board task.
5. Access latest map/procedure PDFs.
6. Submit issue if hazard found.
7. End shift.

### Team leader
1. Start shift.
2. View/edit today board.
3. Submit team production report (one per day).
4. Coordinate via team channel.

### Supervisor
1. Review incidents + high severity.
2. Approve/reject timesheets.
3. View vehicles needing repair.
4. Publish/update board.
5. Review production totals by date/team/site.
